# Quick Task 260723-ox4: Fix mismatched sidebar/top-header heights

**Created:** 2026-07-23

## Problem

User screenshot showed the sidebar's own header block (logo + "Gavel News" +
"Day 3 · 22 Jul 2026 · 4 stories" dateline + collapse toggle) and the main
content area's top header (AppShell.tsx's `<header>`, brand + search +
sign-in) sitting at different heights, so their bottom border lines didn't
align across the page — reads as two mismatched boxes instead of one
continuous top bar.

Root cause: quick task 260723-nwa (nav re-skin) removed the sidebar header's
fixed `h-14` to fit the new two-line dateline, switching it to
content-driven height (`py-2.5`). `AppShell.tsx`'s header was already
content-driven (`py-1.5`, no fixed height) and never matched the sidebar's
height to begin with — the mismatch was subtle before, and became visible
once the sidebar header grew to fit two lines.

## Task

- Files: `components/Sidebar.tsx`, `components/AppShell.tsx`
- Action: give both header rows the same explicit fixed height (`h-16`) with
  `items-center` (already present on both) so content vertically centers
  regardless of whether it's 1 line (mobile AppShell, no-edition-info
  Sidebar) or 2 lines (desktop AppShell with tagline, Sidebar with
  dateline). Replace the vertical-padding classes (`py-2.5` / `py-1.5`) with
  `h-16` on both header containers.
- Verify: `pnpm exec tsc --noEmit`; curl the running dev server and confirm
  both header elements render with the `h-16` class.
- Done: sidebar header and AppShell header are the same height at every
  breakpoint, so their border-bottom lines form one continuous line.

## Must-haves

- truths: sidebar header height === AppShell header height (both `h-16`).
- artifacts: components/Sidebar.tsx, components/AppShell.tsx
