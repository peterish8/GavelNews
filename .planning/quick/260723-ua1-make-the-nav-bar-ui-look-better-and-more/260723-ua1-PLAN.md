# Quick Task 260723-ua1: Give the nav a distinctive legal-editorial identity

**Created:** 2026-07-23

## Context

User feedback: the nav (the actual live nav is `components/AppShell.tsx` + `components/Sidebar.tsx`, driven by `lib/nav.ts`; `components/TopNav.tsx` is confirmed dead/unused code — never imported anywhere, per 260723-nwa's own summary — and stays untouched by this task) "reads as a generic template navbar that any website could have."

This is the third pass at this nav. Prior passes already fixed specific tells:
- **260722-wcu / 260722-wf7:** merged the header into one row, centered the nameplate.
- **260723-nwa:** removed the boxed "Live edition" stat-widget and the repeated mono-uppercase eyebrow (kept it only once, on the 3 section labels), flattened a nested mobile pill-in-pill control. Explicitly decided (user checkpoint) to **keep the sidebar/app-shell structure**, not switch to a conventional top bar.
- **260723-o65:** fixed low-contrast nav item titles (now `text-ink`).

What's left is not a leftover mistake — it's the underlying *component vocabulary*: the nav still uses stock modern-dashboard primitives (filled color pill for the active row, generic icon-in-a-box chips, plain mono-caps section labels) that read the same on a legal-current-affairs reading site as they would on a SaaS admin panel. `app/globals.css` already has a real point of view for this brand ("Warm parchment paper, court-ink type, seal-red brand. Not SaaS/tech glass" — its own comment) — serif display type, IBM Plex Mono for numeric/dateline accents, a `--brand-blend` masthead rule. The nav just isn't using any of it. This task pulls the nav's remaining generic pieces into that already-established identity. Purely visual/markup — no new routes, IA, or state; `lib/nav.ts` and route behavior are unchanged.

## Audit findings (concrete tells to fix)

1. **Filled-pill active state + boxed icon chip in `SidebarLink`** (`components/Sidebar.tsx`): `bg-brand text-on-accent shadow-sm` rounded pill with a `size-8 rounded-lg bg-elevated-muted` icon box is the exact "selected row" pattern used in Linear/Notion/Vercel/Stripe dashboard sidebars. Nothing about it is specific to this site.
2. **`NavIcon` function** (`components/Sidebar.tsx`, ~90 lines): six hand-drawn generic outline glyphs (calendar grid, magnifying glass, bookmark ribbon, gear) plus an unused default circle-question case. Stock icon-library shapes that add bulk without adding brand character.
3. **Section labels have no thematic device**: `SECTION_LABELS` render as bare `font-mono uppercase tracking-[0.16em]` text with no rule/divider and no tie to the site's subject matter (statutes/law).
4. **No masthead rule anywhere in the live nav**: the `--brand-blend` 2px gradient hairline (already defined in `app/tokens.css`/used as a CSS var) exists in code today only inside the dead `TopNav.tsx` (`<div className="h-[2px] w-full" style={{ background: "var(--brand-blend)" }} />`). Neither `AppShell.tsx`'s content header nor `Sidebar.tsx`'s own header shows it, so the one masthead device the codebase already designed for this brand is invisible in the shipped nav.

## Task

### 1. Masthead rule — `components/AppShell.tsx`, `components/Sidebar.tsx`

- In `AppShell.tsx`: the content header is currently a single `<header className="relative flex h-16 shrink-0 items-center gap-2 border-b border-border-app bg-[color-mix(in_srgb,var(--bg)_94%,transparent)] px-2.5 backdrop-blur-xl sm:px-4">` wrapping all header controls directly. Change it to a two-part header: keep `<header>` as the outer element (drop `flex h-16 items-center gap-2` from it, keep the border/background/backdrop-blur/relative), add a `<div className="h-[2px] w-full" style={{ background: "var(--brand-blend)" }} aria-hidden />` as its first child, then wrap the existing row of controls (mobile toggle, collapse toggle, nameplate link, search/sign-in groups) in a `<div className="relative flex h-16 items-center gap-2 px-2.5 sm:px-4">` as the second child. Net effect: a 2px brand-gradient rule sits above the existing 64px control row; total header height becomes 66px (still under the 80px nav-height ceiling).
- In `Sidebar.tsx`: the header block is `<div className="flex h-16 shrink-0 items-center justify-between gap-2 border-b border-border-app px-3">`. Apply the same split: outer `<div className="shrink-0 border-b border-border-app">` containing the `--brand-blend` rule div first, then the existing flex row (minus its own `border-b`, which moves to the new outer wrapper) as a nested `<div className="flex h-16 items-center justify-between gap-2 px-3">`. This gives the sidebar's own header (visible when expanded/mobile-open) the same masthead rule as the top bar, so both nav surfaces share one visual signature.
- Files: `components/AppShell.tsx`, `components/Sidebar.tsx` (header regions only — do not touch `SidebarLink`/`NavIcon`/nav list markup in this step, that's task 2).
- Verify: `pnpm run typecheck` — no type errors; header/sidebar still render one row of controls at their existing height class (`h-16`), rule sits flush above with no gap.
- Done: a 2px `--brand-blend` rule renders at the very top of both `AppShell`'s content header and `Sidebar`'s own header, above the existing control row, in both light and dark mode.

### 2. Docket-index nav body — `components/Sidebar.tsx`

- **Section labels:** in the `sections.map` render, change the label `<p className="mb-1 px-2 font-mono text-[9px] font-semibold uppercase tracking-[0.16em] text-ink-3">{SECTION_LABELS[section]}...</p>` to a serif-italic statute-style label reading `"§ " + SECTION_LABELS[section]` (e.g. "§ Read", "§ Your library", "§ About") using `font-serif italic text-[11px] tracking-[0.01em] text-ink-3`, wrapped so it sits above a `border-b border-border-app/50 pb-1.5 mb-1.5` hairline (replacing the current bare label with no rule beneath it). The "· sign in" suffix on the account section keeps its existing conditional, just inherits the new label styling instead of the mono-caps one.
- **Nav item marker:** delete the `NavIcon` function entirely (all glyph cases + the unused default). In `SidebarLink`, replace the icon box span (`<span className="flex size-8 shrink-0 items-center justify-center rounded-lg ..."><NavIcon name={item.label} /></span>`) with a plain (no background, no rounded box) `<span className="flex size-8 shrink-0 items-center justify-center font-mono text-[13px] font-semibold tabular-nums">` rendering a zero-padded sequential index — compute it as `String(NAV_ITEMS.indexOf(item) + 1).padStart(2, "0")` (passed in as a new `index` prop from the `sections.map`/`items.map` call site, or computed inline in `SidebarLink` via the imported `NAV_ITEMS` — either is fine, keep it a plain derived value, not new state). Color `text-ink-3` normally, `text-brand` when `active && !locked` (no separate background chip). This reuses the mono-digit numbering language already established elsewhere in the app (the sidebar's "Day 42" edition counter, PYQSidebar's "CLAT 2019" mono badges) instead of a stock icon set.
- **Active row treatment:** in `SidebarLink`'s `<Link>` className, replace `active && !locked ? "bg-brand text-on-accent shadow-sm" : "text-ink-2 hover:bg-elevated-muted hover:text-ink"` with `active && !locked ? "border-l-[3px] border-brand bg-brand-soft pl-[calc(0.625rem-3px)]" : "border-l-[3px] border-transparent text-ink-2 hover:bg-elevated-muted hover:text-ink"` (the transparent left border on the inactive branch keeps row width/alignment identical whether or not the border is visible — do not conditionally omit the border utility). Drop the pill's `rounded-xl` on the active branch in favor of `rounded-lg` on both branches (consistent corner radius, no separate active shape). Update the label span's active-state text color from the implicit white (`text-on-accent`) to `text-brand`, and the description subtext's active color from `text-white/75` to `text-ink-2` (no longer needs to contrast against a solid red fill).
- Files: `components/Sidebar.tsx` (the `sections.map`/`SidebarLink`/`NavIcon` regions — do not re-touch the header block edited in task 1).
- Verify: `pnpm run typecheck` — no type errors, no unused imports/exports left over from the deleted `NavIcon`.
- Done: no `NavIcon` function remains; every nav row shows a two-digit mono index instead of a boxed glyph; the active row is a left-rule/tinted tab (not a filled red pill); section labels read "§ <label>" in serif italic with a hairline rule beneath.
