<objective>
Fix two PYQSidebar bugs reported from a live screenshot: (1) markdown syntax in question/option/explanation/passage text renders as literal asterisks instead of bold, because these fields are interpolated as raw strings; (2) clicking a question expands a cramped inline accordion inside the narrow sidebar instead of a proper full-screen detail view.

Purpose: Real ingested CLAT question text contains markdown (e.g. `**1.**`, `**2.**` for numbered sub-statements) that must render correctly, and question detail (passage + question + options + explanation) needs room to be read together, not squeezed into a 320px column.

Output: components/PYQSidebar.tsx renders markdown via the existing `Markdown` component everywhere free text appears, and question rows open a full-viewport modal (built with plain Tailwind, no new dependency) showing the full question detail, replacing the per-row `useState` accordion.
</objective>
